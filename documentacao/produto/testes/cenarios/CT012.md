## Cenário de Teste 012

| **Componente** | **Descrição** |
|----------------|---------------|
| **Identificador** | **CT012** |
| **Nome** | **Registrar agendamento com dados válidos** |
| **Objetivo** | Verificar se o sistema salva corretamente um novo agendamento no banco de dados. |
| **Tipo** | Teste de integração |
| **Dado** | Que o cliente esteja autenticado, selecione uma barbearia existente e informe um horário futuro válido. |
| **Comportamento** | Ao enviar a requisição de criação de agendamento (POST). |
| **Resultados esperados** | O sistema deve persistir o agendamento com status agendado, associado corretamente ao cliente e à barbearia. |
