# REQ001 - Cadastrar Proprietário

**ESCOPO:** Cadastro e gerenciamento do proprietário no sistema.  
**PROPÓSITO:** Permitir que o proprietário crie uma conta com nome, e-mail e senha, garantindo que o e-mail não se repita e possibilitando o acesso à sua área de gerenciamento.  
**ATOR:** Proprietário  

---

### PRÉ-CONDIÇÕES
- O usuário deve acessar a tela de cadastro.
- O e-mail informado não pode estar sendo usado por outro proprietário.

### PÓS-CONDIÇÕES
- O cadastro é feito com sucesso.
- O proprietário é redirecionado para a tela inicial do sistema.
- As informações ficam salvas para uso no login.

---

### FLUXO NORMAL
1. O proprietário acessa a tela “Cadastro de Proprietário”.
2. O sistema mostra os campos de nome, e-mail e senha.
3. O proprietário preenche os campos e confirma o cadastro.
4. O sistema verifica se o e-mail já está sendo usado.
5. Se estiver tudo certo, o sistema cria o cadastro.
6. O sistema mostra mensagem de sucesso e redireciona o proprietário para a tela inicial.

---

### FLUXO DE EXCEÇÃO
- **E1 - Campos obrigatórios vazios:** O sistema mostra mensagem pedindo para preencher os campos.
- **E2 - E-mail já cadastrado:** O sistema mostra mensagem informando que o e-mail já está em uso.
- **E3 - Erro ao salvar:** O sistema mostra mensagem pedindo para tentar novamente.
